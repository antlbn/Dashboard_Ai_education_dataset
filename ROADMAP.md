# Roadmap: дашборд на новом стеке (для новичка)

Цель: быстро собрать интерактивный дашборд по AI Student Impact Dataset и параллельно выучить стек.
Принцип: **один новый инструмент за майлстоун**, учимся через дело, не пытаемся выучить всё сразу.

## Карта стека (зачем каждый инструмент)

| Инструмент | Роль в проекте |
|---|---|
| **Vite** | Сборщик и dev-сервер. Создаёт проект, мгновенный hot-reload. |
| **TypeScript** | Типы. Ловит ошибки до запуска, даёт автодополнение. |
| **Vue 3** | UI-фреймворк. Компоненты, реактивность. Сердце приложения. |
| **Pinia** | Хранилище состояния. Где лежат данные, фильтры, агрегаты. |
| **PrimeVue** | Готовые UI-компоненты (таблицы, дропдауны, карточки). |
| **shadcn-vue** | Набор красивых примитивов (+ Tailwind). Для финальной полировки. |
| **axios** | Загрузка данных по сети (JSON из `public/`). |
| **zod** | Проверка, что данные пришли в правильной форме + типы из схемы. |
| **Chart.js** | Графики (гистограммы, scatter, bar, doughnut). |

---

## M0 — Скелет проекта: Vite + Vue 3 + TS
**Учим:** что такое Vite/npm/node; структуру Vue-проекта; `<script setup>`; реактивность (`ref`, `computed`); компоненты; props/emits.
**Делаем:**
- `npm create vite@latest` → шаблон `vue-ts`, `npm install`, `npm run dev`.
- Разбить экран на layout: `Header`, `Sidebar` (фильтры), `Content` (карточки + графики) — пока пустые компоненты.
- Передать строку-проп из родителя в дочерний компонент, отрендерить.

**Готово, когда:** проект запускается, и ты умеешь создать компонент и передать в него данные.
**Это самый большой кусок обучения — не спеши.**

## M1 — Данные в приложении: axios + zod
**Учим:** `async/await`; axios `GET`; zod-схему; `parse` vs `safeParse`; вывод типов `z.infer<typeof schema>`.
**Делаем:**
- В Python один раз: `df.sample(5000).to_json('public/data.json', orient='records')` (сэмпл, чтобы браузеру было легко).
- Описать Zod-схему одной строки датасета (колонки см. `CLAUDE.md`).
- Через axios загрузить `data.json`, провалидировать массивом, вывести в консоль.

**Готово, когда:** в приложении есть провалидированные **типизированные** данные.

## M2 — Состояние: Pinia
**Учим:** store; `state` / `getters` / `actions`; как использовать store в компоненте.
**Делаем:**
- `dataStore`: хранит строки, статус загрузки (`idle/loading/error`); загрузку из M1 перенести в `action`.
- `filtersStore`: выбранные `Major_Category`, `Year_of_Study` и т.п.
- Getter `GPA_change` (Post − Pre) считать тут, не в данных.

**Готово, когда:** данные грузятся через store, компоненты читают из него.

## M3 — UI-каркас: PrimeVue (shadcn — позже)
**Учим:** установку/настройку PrimeVue, темы, импорт компонентов.
**Делаем:**
- KPI-карточки (`Card`): средний GPA, средние `Weekly_GenAI_Hours`, доля Burnout=High и т.п.
- Фильтры (`Select`/`Dropdown`) по `Major_Category` и `Year_of_Study`, привязанные к `filtersStore`.
- (Опционально) `DataTable` с сырыми строками.

**Готово, когда:** есть визуальный каркас, фильтры меняют состояние в store.

## M4 — Графики: Chart.js
**Учим:** `chart.js` + `vue-chartjs`; регистрацию компонентов; реактивное обновление при смене данных.
**Делаем графики прямо из находок EDA:**
- Гистограмма `Weekly_GenAI_Hours`.
- Bar: средний `GPA_change` по `Major_Category`.
- Scatter: `Weekly_GenAI_Hours` vs `GPA_change` — **подаём честно: связь почти нулевая (r ≈ −0.05), это не «драйвер» GPA** (см. CLAUDE.md).
- Doughnut: разбивка `Burnout_Risk_Level`.

**Готово, когда:** графики перерисовываются при смене фильтров.

## M5 — Интерактивность и связывание
**Учим:** агрегации через `computed`/getters; чтобы не пересчитывать на каждый рендер.
**Делаем:** фильтры → getters в Pinia пересчитывают агрегаты → обновляются и KPI, и графики. Единый живой дашборд.
**Готово, когда:** меняешь фильтр — реагирует весь экран.

## M6 — Полировка и деплой
**Делаем:** адаптивная вёрстка; состояния loading/empty/error (ошибку zod показать пользователю по-человечески); добавить **shadcn-vue** для финального вида; масштабировать данные с сэмпла до полного датасета (или предагрегировать в Python).
**Деплой:** `npm run build` → Netlify / Vercel / GitHub Pages.

---

## Порядок и темп
- Иди строго M0 → M6. Каждый майлстоун — рабочее состояние приложения.
- Не подключай shadcn и PrimeVue одновременно в начале — это два дизайна сразу, для старта много.
- Застрял — спрашивай по конкретному майлстоуну, не пытайся объять всё.
